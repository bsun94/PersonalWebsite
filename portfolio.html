RousseauBot on Twitter
Curious about how Twitter bots worked, I set out to create one myself. The first step is to set up an account for your bot, then apply for Twitter Developers access to that account's back-end. This process should be fairly straightforward and quick within the Twitter Developers portal - just remember to take down all your API/client tokens and keys!

Then it was a matter of grabbing Rousseau source material. I found a webpage on the Online Library of Liberty site containing a work by Rousseau that I liked, On the Social Contract. Using BeautifulSoup and the requests Python libraries, I scraped the webpage for text (using regex to split paragraphs into sentences and to clean the final output e.g. ignore "sentences" that are really chapter titles). This is all then stored in a txt file for a separate script to read.

I wrote another program that read a randomly-selected quote for the txt file above and posted it to Twitter on RousseaBot's account (as shown on the right - go follow!), keeping track of the quotes tweeted over the past 10 days to avoid short-term repetition in a sure-fire manner.

Seeing the spread of covid-19
As we lived through more and more of the pandemic, I noticed that cases and deaths were mostly concentrated in a few countries (I'd kept track of online COVID counts). That sparked the idea to build a tracker that can display a choropleth map of COVID cases and deaths around the world.

Finding an online data source was the most important first step. Thankfully, the folks at the Our World in Data offers data pulled from the European Centre for Disease Control for free use.

The program I wrote would first take user input for whether they want to see cases or deaths data visualized, and for which date. Based on these inputs, it would then look up the relevant data from the OWID data source above and clean it up (e.g. for inconsistencies in country name spellings). It then builds the choropleth maps using the folium library, and massages pieces of the COVID data for presentation on the map. With selenium, it finally outputs an interactive map on the user's screen.

twitter's much younger counsin
I'd always been curious if I could build a rudimentary version of a social media platform - just as a proof-of-concept for myself to see how it could all work. Aware that I do not have the same kind of budget that Facebook and Twitter have, and that I'll be programming this in my spare time outside of a full-time job, I decided to take a tongue-in-cheek approach to this project - calling it a "Low-Budget" Twitter.

The application is capable of handling user logins (checking if username already exists before registration; validating inputted username and password - the login screen is shown in the left image), inputting and display of tweets on a main feed (middle image) and build conversation chains/rooms for each main feed tweet (i.e. all tweets belonging to a particular conversation thread - example shown in the image to the right). The logo drawn for the application, as can be seen on the login window, is meant to reflect the tongue-in-cheek side of this project (and avoid copyright) :)

I used the tkinter module in Python to build out the UI for this application. I'd learnt about tkinter ever when I first learnt Python, and was always keen on using it to build something. Admittedly, tkinter doesn't necessarily make for the most modern-looking social media app - I was more so curious and keen to get deeper into the module.

As for the database side of things, I set up a free account with IBM to use their Lite database plan. The script makes a connection to the IBM database when storing tweets/replies that users post, as well as user login information. For security purposes, user passwords are hashed prior to database storage. The overall structure of the code is based on the MVC design pattern.
