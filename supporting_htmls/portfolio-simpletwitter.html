twitter in two weeks!
I'd always wanted to build a rudimentary version of a social media platform - just as a proof-of-concept for myself to see how it could all work. Aware that I do not have the same kind of budget that Facebook and Twitter have, and that I'll be programming this in my spare time outside of a full-time job, I decided to take a tongue-in-cheek approach to this project - calling it a "Low-Budget" Twitter.

The application is capable of handling user logins (checking if a username already exists before registration; validating username and password inputs - the login screen is shown in the left image), the inputting and display of tweets on a main feed (middle image) and building conversation threads for each main/original tweet and its replies (and subsequently displaying individual conversation threads in the window - example shown in the image to the right). The logo drawn for the application, as can be seen on the login window, is meant to reflect the tongue-in-cheek side of this project (and to avoid copyright). :)

I used the tkinter module in Python to build out the UI for this application. I'd learnt about tkinter when I first learnt Python, and was always keen on using it to build something. Admittedly, tkinter doesn't necessarily facilitate the most flashy modern-day visual designs - I was more so curious and keen to get deeper into the module.

As for the database side of things, I set up a free account with IBM to use their Lite database plan. The script makes a connection to the IBM database when storing tweets/replies that users post, as well as user login information. For security purposes, user passwords are hashed prior to database storage. The overall structure of the code is based on the MVC design pattern.
